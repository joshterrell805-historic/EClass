 11     """ 
 12     A Presentation is a collection of slides and their respective layers which is displayed at  
 13     the center of the EClass. View the Slide documentation for more detailed information about them. Presentation contains a variety of methods used to navigate between slides 
 14     and access the current slide. It handles all of the model-related syncing functionality between presenters and students as well. 
 15     It accomplishes this by passing messages back and forth between the presenter and the student(s). On a presenter client, the current slide is broadcast 
 16     to each student whenever the slide is changed. A student's Presentation object will listen for the message and update accordingly if it is  
 17     currently set to stay synced. 
 18      
 19     Presentation also includes the Slidify() method which opens up the HTML file located at a Presentation's 
 20     path and converts it to a fresh set of slides to be displayed. Any <br class="slide"> tags 
 21     in the HTML denote the location where a new Slide should be created and Slidify() will break  
 22     them up at those points. However, the method ensures there will always be at least one slide if 
 23     there are no <br class="slide"> tags. 
 24   
 25     @author: Joel Wilcox (jnwilcox@calpoly.edu), Kevin Le (kle17@calpoly.edu) 
 26   
 27     @ivar currSlideNum: The index of the current slide. 
 28     @ivar slides: Contains each of the slides in a presentation. 
 29     @ivar path: Refers to the current file system path of the backing presentation file. 
 30     @ivar isSynced: Boolean to keep track of whether the slides should be synced with the presenter's slides. 
 31     @ivar rawHTML: A string copy of the presentation HTML contents.

261     """ 
262     A layer is a collection of layer objects. Layers are like a sheet of paper 
263     on an overhead projector. You can stack them on top of eachother, swap 
264     positions, and draw on whichever one you want. They are always drawn in 
265     a specified order. Layers have several different properties including: 
266     the visual objects each layer contains, opacity (determining 
267     how transparent to draw each object), visiblility (a toggle 
268     determining whether to draw the layer at all), and lock (a toggle determing 
269     whether objects may be added to or moved on the layer). 
270      
271   
272     Layers contain layer objects which are python dictionaries holding 
273     representations of visual objects. The currently implemented layer objects are: 
274     square, text, and pencil. Each object has a type property defining what type it is 
275     "square" "text" or "pencil" and then a varying set of properties determining 
276     how to draw the object (position, points, width, height, ...) 
277   
278     @author: Andrew Lisowski (alisowsk@calpoly.edu) + Josh (class doc) 
279   
280     @ivar objects: A Collection of LayerObjects that represent all graphics on a Layer 
281     @ivar opacity: A double representing how transparent a Layer is 
282     @ivar visible: Boolean value representing if a Layer can or cannot be seen 
283     @ivar permissions: The LayerPermissions for a given Layer
