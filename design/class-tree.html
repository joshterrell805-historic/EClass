Class Hierarchy

  • EClass.ApprovalTracker.ApprovalRating: ApprovalRating is a value which represents the approval of the presentation at this very instant.
  • EClass.Connection.Connection: Connection class is a singleton which represents the connection between the user and everything external.
  • EClass.View.DrawingTools: A DrawingTools is a tool bar containing the various drawing tools a student or presenter may use during a presentation.
  • EClass.Presentation.DrawingToolsModel: A DrawingToolsModel holds the functionality of selecting any of the drawing tools from the DrawingTools tool bar.
  • EClass.EClass: EClass is a collection of all the highest level models in the EClass program including (but not limited to) the presentation, layer manager, and logged in user.
  • EClass.View.EClassWindow: EClassWindow is the main window for the EClass program.
  • EClass.Forum.Forum: A Forum contains all of the messages from user to user and methods to manage the forum.
  • EClass.Presentation.Layer: A layer is a collection of layer objects.
  • EClass.View.LayerManager: Renders the view of the LayerManagerModel
  • EClass.Presentation.LayerManagerModel: This model manages the layers attached to the current slide.
  • EClass.View.LayerView: Renders a Layer inside the LayerManager.
  • EClass.View.MenuBar: MenuBar is a collection of all menu operations including file operations and view operations that hide/show various windows.
  • EClass.Forum.Message: A Message contains all of the information for a message including the user's name, the datetime that a message was sent, and the text of the message.
  • EClass.View.NewLayerWindow: Renders the dialog for a New Layer.
  • EClass.Presentation.PermissionLevel: A PermissionLevel is an Enum of the three presentation permission levels used for students.
  • EClass.Person.Person: Person is a class that is instantiated when someone logs into the EClass.
    • EClass.Person.Presenter: Presenter contains information specific to the presenter.
    • EClass.Person.Student: Student contains informaton on the student.
  • EClass.Presentation.Presentation: A Presentation is a collection of slides and their respective layers which is displayed at the center of the EClass.
  • EClass.Person.Roster: A Roster contains all of the students in the class and methods to manage the students.
  • EClass.Person.RosterItem: A RosterItem contains an individual student's first and last name as long as buttons for student question asking, pushing layers, and permission setting.
  • object: The most base type
    • EClass.Connection.Class
      • EClass.Connection.PresenterClass: The type of class in the classes array if the logged in user is a presenter.
      • EClass.Connection.StudentClass: The type of class in the classes array if the logged in user is a student.
    • EClass.Connection.GenericResponse: A generic response can either be a success or a failure.
      • EClass.Connection.AuthenticationResponse: This is the response received in the callback when authenticating.
      • EClass.Connection.GenericFailure: A generic failure is a GenericResponse that failed.
        • EClass.Connection.AuthenticationFailure
        • EClass.Connection.HostFailure
      • EClass.Connection.GenericSuccess: A generic success is a GenericResponse that succeeded.
        • EClass.Connection.AuthenticationSuccess
        • EClass.Connection.HostSuccess
      • EClass.Connection.HostResponse: Response passed to the callback when attempting to host.
    • EClass.Connection.JoinResponse: Response passed to the callback when attempting to join.
      • EClass.Connection.JoinFailure
      • EClass.Connection.JoinSuccess
    • wx._core.Object: The base class for most wx objects, although in wxPython not much functionality is needed nor exposed.
      • wx._core.EvtHandler: Proxy of C++ EvtHandler class
        • wx._core.Window: wx.Window is the base class for all windows and represents any visible object on the screen.
          • wx._windows.Panel: Proxy of C++ Panel class
            • EClass.View.InitialPrompt: InitialPrompt represents the panel that the user first sees to either create or open a new presentation.
            • EClass.View.RosterItemPanel: RosterItem is the panel that displays a student's first and last name along with three buttons for question asking, pushing layers, and student permissions.
            • EClass.View.RosterStaticPanel: The static panel is the panel that is the parent panel to each RosterItemPanel that displays when a student is selected.
            • EClass.View.WhiteboardNav: WhiteboardNav defines the view for the whiteboard (where slides are displayed) as well as the navigation controls for changing the current slide.
          • wx._windows.TopLevelWindow: Proxy of C++ TopLevelWindow class
            • wx._windows.Frame: Proxy of C++ Frame class
              • EClass.View.ChangePermWindow: Shows the Dialog to change the permissions on a layer.
              • EClass.View.ForumWindow: Forum is the window that is shown when the user presses the 'Show Forum' menu item from the 'View' menu.
              • EClass.View.ImportPresentation: A ImportPresentation is a window that allows the importing of an already made HTMLpresentation document
              • EClass.View.KickWindow: KickWindow defines the window used to confirm whether or not a student will be kicked.
              • EClass.View.LoginWindow: LoginWindow is the window that is shown when the application is first launched.
              • EClass.View.PermissionsWindow: PermissionsWindow defines the window used to display and access a student's permissions.
              • EClass.View.RosterWindow: Roster is the window that is shown when the user presses the 'Show Roster' menu item from the 'View' menu.
    • EClass.Person.Question: Question is a container for the text and time created of a Student's question.
    • EClass.Presentation.Slide: A Slide is a collection of presentation content and layers that can be edited by a presenter or student.
    • EClass.Connection.Student
    • EClass.Person.StudentPermissions: StudentPermissions has various access rights to tools that the student can use.
