Student Acceptance Testing


Case No. User Inputs and Commands Expected Output Remarks                   Results                  
1 (As a presenter) From the MenuBar->View select 'Show Roster'. With the Roster Window open, click on a student and click the 'Permissions' button. The Permissions dialog pops up with the student's current permissions indicated by the radio buttons and check boxes. Testing opening the the student permissions dialogue and accessing student permissions. Pass
2 (As a presenter) With the permissions dialog open, change the student's whiteboard level by selecting the 'Lockdown' radio button and click 'Accept' On the student's client, they will see a pop-up notifying them that they have been placed in lockdown mode and have one minute before it takes effect. After a minute, the student will not be able to use any whiteboard navigation controls and will be synced with the presenter. Testing the 'Lockdown' student permission and changing student permissions via the dialog. Pass
3 (As a presenter) With the Roster Window open, click on a student and click the 'Kick' button. The Kick confirmation dialog pops up. Testing opening the Kick dialog. Pass
4 (As a presenter) With the Kick dialog open, click 'Yes'. The student will receive a pop-up notifying them that they have been kicked but will still be able to edit their layers and browse the slides if they want to. The roster window is updated so that the student is now absent and they will no longer be able to interact with the presenter or other students. Testing the the ability to kick students. Pass
5 (As a student) Close and reopen EClass after being kicked. Login and attempt to join the presentation you were kicked from. The window will display an error letting the student know that they have been kicked and are not allowed to rejoin the presentation. Testing that a student's kicked status is preserved even after they close their instance of EClass. Pass
