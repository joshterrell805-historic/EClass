HOW TO RUN EClass




  1. Make sure Python version 2.7.X is installed. This is the version our team used and we know wxPython is supported in
  2. Make sure wxPython version 3.0 is installed. This is the version our team used and we know is supported by Python 2.7.X
  3. Make sure twisted is installed https://twistedmatrix.com/trac/wiki/Downloads
  4. Once twisted is installed, make sure twisted install path is on your PYTHONPATH. It's in /lib/.... do a find to find it
    1. `sudo find / -name twisted`
    2. `export PYTHONPATH=$PYTHONPATH:/System/Library/Frameworks/Python.framework/Versions/2.7/Extras/lib/python` is what worked on Andrew's mac
    3. NOTE: it's not ../lib/python/twisted ... it's .../lib/python
  5. Make sure webkit is installed (should be pre-installed on mac)
  6. The path where the program is to be ran from is: http://users.csc.calpoly.edu/~kle17/EClass/implementation/source/python/
  7. In order to log in, the central server must be running. To run the central server, run python main.py server from a separate terminal
  8. To run the program simply run the command: python main.py
  9. To authenticate, your username is `gfisher` and no password is required.
  10. It is now possible to launch a student user that is connected with the presenter through the central server. To do this, open up another instance of EClass and login with the username 'student'. From the Professor terminal, a print message will display the user joining and when they leave as well.
  11. 'student' can only join a presentation when the professor is hosting one.

For convenience, here is a link to our documentation: EClass Doc
Also a note for the documentation, @author tags are only included in the documentation files and not the source files. We discussed this with you in a meeting during class.

NOTE: If you run EClass on Windows, there is occasionally an issue with student syncing where their whiteboard will not sync up with the presenter every time the slide is changed due to our need for using wx's version of Sleep(). It's possible this may occur on Mac/Linux as well but we have not encountered it on those platforms yet.

While EClass works pretty well in the three major operating systems (Linux, Mac, Windows), they each ultimately have some unique quirks when interacting with wxPython. If you run into something strange (you shouldn't, but if you do...) please see if it occurs on one of the other operating systems as it may be an issue with only one platform.
